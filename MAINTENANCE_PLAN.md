# Maintenance Plan: Micronaut Petclinic (Official Sample)

This document defines how the Micronaut Petclinic sample is kept in sync with Micronaut releases and how releases are published automatically.

## Goals
- Keep the sample updated with each Micronaut release.
- Validate changes automatically (build/tests).
- Publish a matching release tag automatically once updates are merged.

## Workflow Overview

**Micronaut release → Version update PR → CI validation → Merge → Release published**

## 1) Automated Version Sync

**Trigger:** New Micronaut release

**Mechanism (implemented):**
- Use Renovate (`.github/renovate.json`) to open PRs for dependency and Micronaut parent updates.
- Auto-merge patch updates; require manual review for minor/major updates.

**Expected Output:**
- A PR titled like: `chore(deps): update Micronaut to <version>`

## 2) CI Validation (Required on Every PR)

**Checks:**
- Build
- Test suite
- Static checks/linting (if present)

**Result:**
- PRs only merge once CI is green.

## 3) Automated Release (On Merge)

**Trigger:** Push to default or release branch where `gradle/libs.versions.toml` changed

**Mechanism (implemented):**
- A GitHub Action (`.github/workflows/release.yml`) validates version consistency and creates:
  - a git tag matching the Micronaut version (e.g., `v4.2.1`)
  - a GitHub Release with autogenerated notes

**Notes:**
- If the repository already uses a versioning scheme, follow it consistently.
- Consider a suffix if needed (e.g., `v4.2.1-petclinic.1`).

## Ownership & Cadence

**Owners:**
- Sample maintainers own merges and release approvals.

**Cadence:**
- Automated PRs appear immediately after Micronaut releases.
- Maintainers review/merge once CI passes.

## Failure Handling

If an update PR fails CI:
- Investigate failures (breaking changes, deprecated APIs, etc.).
- Fix in the PR and re-run CI.
- Do not release until all checks pass.

## Optional Enhancements

- Add a nightly workflow to verify the sample builds against the latest Micronaut snapshot.
- Add a manual workflow dispatch to trigger a release if needed.
- Pin plugin versions if upstream introduces breaking changes.

## Quick Checklist

- [ ] Automated Micronaut version PR created
- [ ] CI is green
- [ ] PR merged to default branch
- [ ] Git tag created
- [ ] GitHub Release published
